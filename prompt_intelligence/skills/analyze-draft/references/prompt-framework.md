# Prompt Structuring Framework

Use these five dimensions when analyzing or building a prompt:

| Dimension | Question to answer |
| --- | --- |
| **Role** | Who should the model act as? |
| **Task** | What specific action must it perform? |
| **Context** | What background, data, or situation applies? |
| **Constraints** | What limits apply (length, tone, exclusions, format rules)? |
| **Output format** | What shape should the response take (bullets, JSON, sections)? |

A strong prompt makes each dimension explicit. Vague drafts usually omit one or more of these.
